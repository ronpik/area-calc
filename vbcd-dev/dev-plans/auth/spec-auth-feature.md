## 1. Overview

### 1.1 Feature Summary

Add user authentication to AreaCalc, enabling users to:
- Sign in with Google account (single-click authentication)
- Save measurement sessions to Firebase Cloud Storage
- Access saved sessions from any device
- Manage sessions (view, load, rename, delete)
- Delete their account and all associated data

### 1.2 Key Decisions

| Decision | Choice |
|----------|--------|
| Auth Provider | Firebase Auth with Google Sign-In |
| Sign-in Method | Popup window |
| Storage Backend | Firebase Cloud Storage |
| Auth Button Location | Floating button, top-right corner |
| Language Support | Device locale detection (English/Hebrew) |
| Session Limit | Unlimited |
| Local Data on Login | Auto-save as new session |
| Session Naming | Prompt with auto-generated default |
| Load Behavior | Replace current points (with confirmation) |

### 5.2 Firebase Initialization

```typescript
// src/lib/firebase.ts

import { initializeApp, getApps } from 'firebase/app';
import { getAuth, GoogleAuthProvider } from 'firebase/auth';
import { getStorage } from 'firebase/storage';

const firebaseConfig = {
  apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
  authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
  projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
  storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
  appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
};

// Prevent multiple initializations
const app = getApps().length === 0 ? initializeApp(firebaseConfig) : getApps()[0];

export const auth = getAuth(app);
export const storage = getStorage(app);
export const googleProvider = new GoogleAuthProvider();
```

### 5.3 Firebase Storage Security Rules

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // Users can only access their own data
    match /users/{userId}/{allPaths=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Deny all other access
    match /{allPaths=**} {
      allow read, write: if false;
    }
  }
}
```

## 11. Security

### 11.1 Authentication Security

- **Token handling:** Firebase SDK manages tokens automatically
- **Token refresh:** Automatic (Firebase handles expiration)
- **Sensitive data:** No sensitive data stored client-side beyond Firebase auth state

### 11.2 Storage Security

- **Access control:** Firebase Storage rules restrict access to user's own data
- **Data isolation:** Users cannot access other users' data
- **HTTPS:** All Firebase communication over HTTPS

### 11.3 Client-Side Considerations

- **localStorage:** Contains only session points (not sensitive)
- **No credentials:** User credentials never handled by app (delegated to Firebase/Google)
- **Environment variables:** Firebase config is public (by design) - security via rules

## 12. Edge Cases

### 12.1 Concurrent Sessions (Same User, Multiple Devices)

**Scenario:** User has app open on phone and tablet simultaneously.

**Behavior:**
- Each device operates independently
- No real-time sync between devices
- Last write wins for index.json and individual sessions
- User should refresh "My Sessions" to see updates from other devices

**Mitigation:** Add "Pull to refresh" or "Refresh" button in My Sessions modal.

### 12.2 Session File Missing but in Index

**Scenario:** index.json lists a session, but the session file was deleted (corruption, manual deletion).

**Detection:** When user tries to load, fetch returns 404.

**Behavior:**
1. Show error: "Session not found"
2. Offer to remove from list
3. If user confirms, update index.json to remove the orphaned entry

### 12.3 Index File Corruption

**Scenario:** index.json is malformed JSON or has invalid structure.

**Detection:** JSON parse error or schema validation failure.

**Behavior:**
1. Log error for debugging
2. Show error state in My Sessions: "Failed to load sessions"
3. Offer "Reset" option that deletes and recreates index.json
4. Note: Individual session files are preserved

### 12.4 Very Long Session Names

**Handling:**
- Input max length: 100 characters
- Display truncation: Ellipsis after ~30 characters in list view
- Full name shown on hover/focus (tooltip)

### 12.5 Empty Session (0 Points)

**Prevention:** "Save" button disabled when points.length === 0

### 12.6 Rapid Save Clicks

**Handling:**
- Disable Save button immediately on click
- Show loading spinner
- Re-enable only after operation completes (success or failure)

### 12.7 Offline Usage

**Behavior:**
- App works fully offline (existing functionality)
- Auth operations fail gracefully (show network error)
- Save/Load operations fail gracefully (show network error with Retry)
- localStorage continues to work as primary storage

### 12.8 Firebase Quota/Limits

**Awareness:**
- Firebase Storage free tier: 5GB storage, 1GB/day download
- For individual user: Effectively unlimited for text JSON files
- Monitor usage in Firebase Console
